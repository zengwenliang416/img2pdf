# 1. 竞品工作流拆解

## 核心流程模型

以扫描全能王为参照，核心流程：

```
输入 → 处理 → 编辑 → 导出 → 管理
```

## 详细工作流对比

| 阶段     | 扫描全能王工作流                              | MVP 对标                      |
| -------- | --------------------------------------------- | ----------------------------- |
| **输入** | 相机实时取景 + 相册导入 + 批量选择            | 相机拍照 + 文件上传 + 批量    |
| **处理** | 自动边缘检测 → 透视矫正 → 裁剪                | OpenCV.js 边缘检测 + 四点调整 |
| **编辑** | 滤镜（原色/增强/灰度/黑白/去阴影）+ 旋转/裁剪 | 3种滤镜 + 去阴影 + 旋转       |
| **导出** | PDF（单页/多页）+ JPG + 分享链接 + OCR文字    | PDF/JPG + 可选OCR/分享        |
| **管理** | 文件夹分类 + 搜索 + 云同步 + 回收站           | 列表 + 搜索 + 本地存储        |

## 差异化优势

1. **纯Web端**：无需下载App，跨平台兼容
2. **本地优先**：默认不上传，保护隐私
3. **开源透明**：代码可审计，无后门担忧
4. **轻量快速**：无广告，无付费弹窗

## 用户场景分析

### 场景1：办公文档扫描

- **用户**：白领、学生
- **痛点**：需要快速将纸质文档电子化
- **解决**：拍照即扫描，一键导出PDF

### 场景2：票据归档

- **用户**：财务人员、报销人员
- **痛点**：发票、收据需要存档
- **解决**：批量扫描，多页合并

### 场景3：书籍笔记

- **用户**：学生、研究人员
- **痛点**：书本内容需要电子化保存
- **解决**：去阴影增强，OCR提取文字
